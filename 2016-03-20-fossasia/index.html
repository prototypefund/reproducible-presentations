Can we trust this process?

 

Reproducible builds

  • Ensure compilation always produces the same result
  • Multiple parties compare results
  • Attacker needs to infect everything simultaneously
  • Bit-for-bit identical

Can we now really trust what is running on your computers?

Challenges

  • Timestamps
  • Timezones & locales
  • Different versions of libraries
  • Non-deterministic file ordering
  • Users, groups, umask, environment variables
  • Random behaviour (eg. hash ordering)
  • Build paths

Technical advantages

  • Unsafe/unreliable behaviour (eg. internet access)
  • Non-deterministic behaviour
  • Being able to "go back in time"
  • Detect corrupted build environments
  • Easier to test changes/revisions
  • Continuously testing archive
  • 7 amd64 machines, 120 cores, 300 GB RAM, 15 armhf nodes
  • 20,079 / 23,595 packages are reproducible
  • Still need some core changes
